# 方法论文档

> 架构背后的"为什么"和"怎么做"——不是规则本身，而是规则存在的原因。

如果你在找**编码规则**（什么能做/什么不能做），请看 [`CLAUDE.md`](../../CLAUDE.md)。
如果你在找**架构定义**（每一层/每个 artifact 是什么），请看 [`knowledge_base/architecture/`](../architecture/)。
如果你在找**Artifact 模板**（每个文件应该长什么样），请看 [`knowledge_base/artifacts/`](../artifacts/)。

## 阅读建议

- **刚接触本项目？** 从 `架构最佳实践` 开始——它解释了你在代码中遇到的每一个设计决策背后的动机。
- **对团队工作流好奇？** 读 `任务雷达`——它讲述了这套架构如何让团队无摩擦协作的故事。
- **在构建 AI 辅助项目？** 读 `Harness 工程`——它解释了如何组织代码仓库让 AI Agent 有效导航。
- **思考未来方向？** 读 `Agent 并行开发`——它描绘了从人类团队到人机混合团队的路径。

## 原则

这里的每篇文档遵循一条规则：**一个文档，一个问题。** 没有文档试图回答"X 是什么"——架构和 artifact 文档已经做了这些。这里的文档回答"为什么是 X"和"怎么思考 X"。

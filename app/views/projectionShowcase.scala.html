📡 How events move

Unlike Kafka or RabbitMQ, this project uses a Pull-based mechanism. The Projection service polls the MongoDB journal for new events using Akka Persistence Query.

🆔 Instance Identifier

The ID next to 'Monthly Summary' is the Projection Instance ID. It tracks the physical Offset in the DB, ensuring events are processed exactly once.

🛡️ Domain Invariants

All business rules (like 'Min Balance') are checked on the Write Side (Aggregates). The Read Side only reflects results after a successful state transition.

架构一览

系统遵循洋葱架构，具有严格的依赖方向规则。 领域核心是纯 Scala 代码——不依赖任何框架，不执行 I/O。应用层编排领域逻辑。基础设施层对接外部系统。 这是高吞吐量金融系统中常见的架构模式。

基础设施层 — Play 控制器、MongoDB 事件日志、MySQL 读库、Protobuf 序列化
▲ 依赖注入
应用层 — 聚合根 (EventSourcedBehavior)、Saga 事务协调器、投影处理器、应用服务
▲ 依赖
领域层 — 实体、值对象、领域事件、不变量规则（零外部依赖）
MongoDB — 事件日志（写） MySQL — 投影存储（读） Akka 集群分片 Akka 持久化 Akka 投影

反应式宣言

2014 年由 Akka 创始人 Jonas Bonér 发布， 《反应式宣言》 定义了构建弹性、灵敏、可伸缩系统的四大基石。 Akka 从设计之初就完全贯彻了这些理念。

📨

消息驱动

异步消息传递是系统的骨架。松耦合、位置透明、非阻塞通信支撑了弹性和伸缩性。

Akka：Actor 模型的异步消息传递
🛡️

韧性

故障被隔离在单个 Actor 内部。监督层级树控制故障传播，系统自我修复，不会产生雪崩效应。

Akka：监督策略、断路器、退避重试
📈

弹性

根据负载动态伸缩。集群分片将实体分布到多个节点，自动重新平衡，无需手动干预。

Akka：集群分片、自适应负载均衡
⚡

灵敏性

在任何条件下都提供快速、一致的响应时间。非阻塞 I/O 和背压机制确保高负载下延迟仍然可控。

Akka：全链路非阻塞、Reactive Streams 背压

事件溯源 × CQRS

事件溯源和 CQRS 是互补的模式。结合反应式原则，它们构成了一个强大的系统基础， 适用于需要可审计性、可伸缩性和高韧性的场景。Akka 为两者提供了开箱即用的一等支持。

事件溯源

系统状态通过重放不可变的事件日志来重建。每一次状态变更都被捕获为领域事件——提供完整的审计追踪、时间点查询和可靠的调试能力。Akka Persistence 自动处理事件日志和快照。

CQRS — 命令查询职责分离

命令修改状态（写端，事件溯源）。查询读取优化的投影（读端，MySQL）。这种分离让两端可以独立伸缩，各自使用最适合的数据模型。

实时投影（ETL 流处理）

Akka Projections 将事件流转换为读模型表。每个投影订阅标签事件，事务性地处理它们并更新 MySQL——提供最终一致性的查询模型，保证至少一次投递。

Saga 模式（TCC 分布式事务）

基于 Try-Confirm/Cancel 模式的分布式事务。每个步骤都有 Prepare → Commit → Compensate 三个阶段。失败触发向后补偿。挂起的事务支持人工介入。全部由事件溯源的 Saga Actor 协调。

能力维度传统 CRUD事件溯源 + CQRS
审计追踪手动日志，不完整每次状态变更都是事件——完整历史
读性能被写操作竞争专用读模型，独立伸缩
调试"下午2点的状态是什么？"——很难回答重放事件到任意时间点
模式演进迁移是破坏性的事件只追加；上抛器处理演进
系统集成批量 ETL，轮询实时事件流，推送式
故障恢复重试循环、死信队列监督层级 + Saga 补偿

免费部署资源指南

本项目需要 MongoDB（事件日志）+ MySQL（读模型）+ JVM 应用服务器。 以下是真正提供免费套餐的平台，可以跑起来完整的技术栈。

Fly.io

免费 — 3 台共享 VM（每台 256MB）

将 Play 应用打包为 Docker 容器部署。全球边缘网络。适合跑应用服务器。

MongoDB Atlas

免费 — M0 集群，512MB 存储

共享集群用于事件日志。demo 级别完全够用。内置监控面板。

PlanetScale

免费 — 1 个数据库，5GB 存储

MySQL 兼容，免费额度很充裕。适合读模型。支持数据库分支，方便变更管理。

Render

免费 — Web 服务 + PostgreSQL

GitHub 一键部署。Web 服务闲置后会休眠（有冷启动延迟），但用作 demo 足够。

Oracle Cloud 永久免费

免费 — 4 核 ARM，24GB 内存 VM

最慷慨的免费计算资源。用 Docker Compose 在一台机器上跑全套。

Railway

每月 $5 额度，无时间限制

一键部署，内置 MySQL 和 MongoDB 模板。比 Fly.io 更简单的 UX。

推荐组合方案

Fly.io（应用服务器）+ MongoDB Atlas M0（事件日志）+ PlanetScale（读模型）。 三个平台都有不会过期的免费套餐。总费用：$0/月。 只需要一份 fly.toml 加上几个环境变量配置连接字符串即可部署上线。
